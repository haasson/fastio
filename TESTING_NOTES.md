### 3.2.2 Блюда — фото по URL
- 🐛→🔧 URL-загрузка изображений не работает для западных CDN (Unsplash, Pixabay, Google Photos и др.)
  - **Причина (перепроверено 2026-06-12 с VPS):** НЕ TLS. `curl` до images.unsplash.com / cdn.pixabay.com → `time_connect=0.000000` + таймаут = **TCP-коннект не устанавливается** (пакеты не доходят). Общий исходящий HTTPS жив (picsum/httpbin коннектятся за 0.3с). Вывод: западные CDN на Fastly/Cloudflare **дропают коннекты с российского IP** VPS — гео-блок, в коде/простым прокси не лечится.
  - **Влияние на юзера — мелкое:** ломается только путь «по URL», файл-аплоад работает всегда; сток-URL для реального меню почти никто не вставляет. Реальный вред = плохой UX на редком пути.
  - **🔧 Сделано (`proxy-image.ts`):** на сетевом провале (500/502/504 + клиентские фейлы) сообщение «Не удалось загрузить по ссылке. Скачайте картинку и загрузите файлом с компьютера». Гео-блок не побеждали — направили на рабочий путь. ⚠️ admin-ребилд.
  - **Не делали:** egress-прокси вне РФ (настоящий инфра-фикс) — парк. Edge-таймаут 10с не трогали (юзер просил только сообщение).
