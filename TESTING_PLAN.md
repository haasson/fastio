# FastIO - Manual Testing Plan (Production)

> Дата: 2026-04-08
> Цель: полное ручное тестирование всех модулей, edge cases и межмодульных взаимодействий

---

## Условные обозначения

- **[HP]** — Happy Path (основной сценарий)
- **[EC]** — Edge Case (граничный случай)
- **[NEG]** — Negative Test (ожидается ошибка/отказ)
- **[CROSS]** — Cross-module (взаимодействие модулей)
- **[RT]** — Realtime (проверка живых обновлений)

Приоритеты:
- P0 — критичный (блокирует работу)
- P1 — важный (влияет на основной flow)
- P2 — средний (неудобство, но не блокер)
- P3 — низкий (косметика, редкие кейсы)
