# delivery — заметка для агента

Витрина доставки еды (retail). Не путать с `appointments` (запись на услуги) — здесь только про доставку блюд.

## Что модуль делает

Страница `/delivery` показывает гостю:
- интерактивную карту с зонами доставки (Yandex Maps);
- текстовое описание условий для каждой зоны (стоимость, минимум, время);
- общий блок «Доставка от X ₽» сверху.

Сами зоны хранятся в таблице `delivery_zones` (админка управляет), читаются через `/api/delivery-zones`. Карта рендерится через `DeliveryMapView.vue` (async-loaded, тяжёлый компонент с Yandex Maps).

## Карта модуля

| Файл | Что внутри |
|---|---|
| `utils/deliveryText.ts` | `buildDeliveryText` (общий блок «от X ₽», берёт самую дешёвую зону) + `formatZoneConditions` (детальное описание одной зоны) |
| `components/DeliveryMapView.vue` | Карта Yandex с полигонами зон + drawer с условиями выбранной зоны |
| `__tests__/deliveryText.test.ts` | Юнит-тесты на форматтеры |

## Типовые задачи

- **Новое условие зоны (например, бесплатная доставка от X):** правь `formatZoneConditions` в `utils/deliveryText.ts` + добавь тест.
- **Изменить логику «от X ₽»:** правь `buildDeliveryText` — он ищет самую дешёвую активную зону.
- **Кастомизация карты (стиль, маркеры):** правь `components/DeliveryMapView.vue` — это async компонент.

## Антипаттерны (не делай так)

- ❌ Импорт `~/features/delivery/utils/deliveryText` снаружи модуля — через `~/features/delivery` (barrel).
- ❌ Грузить `DeliveryMapView` синхронно — Yandex Maps SDK тяжёлый (~300 KB), всегда `defineAsyncComponent`.
- ❌ Считать минимум/стоимость доставки на клиенте при чекауте — сервер делает финальный расчёт через зоны. На клиенте только display.

## Куда расти

- Бесплатная доставка от суммы (см. WISHLIST: «Free delivery threshold»).
- Тарифы доставки по времени суток (см. WISHLIST: вечер/ночь наценка).
- Интеграция с Яндекс Доставкой (см. WISHLIST: б2б-курьер вызвать) — это уже про логистику, не про витрину.
