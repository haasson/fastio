## Команда
- **URL:** /team
- **Пермишен:** team.view (просмотр), team.manage (управление), roles.manage (роли)

### Сотрудники
- **URL:** /team/members
- **Действия:**
  - Секция «Пригласить в команду»: поле Email, выбор роли, доступ к филиалам (все или конкретные) → кнопка «Пригласить»
  - Секция «Участники»: список сотрудников (имя, email, роль, филиалы)
  - Редактировать: сменить роль, изменить доступ к филиалам
  - Заблокировать/разблокировать сотрудника
  - Удалить из команды
  - Секция «Ожидают принятия»: кнопка «Отменить приглашение»
- **Пустое состояние:** «Пока нет участников»
- **Валидация:** «Введите email», «Некорректный email», «Выберите роль»

### Роли
- **URL:** /team/roles
- **Модуль:** customRoles (для создания своих ролей)
- **Действия:**
  - Кнопка «Создать роль» → форма (название, настройка пермишенов по категориям: меню, заказы, столы, бронирования, кухня, акции, контент, команда, роли, настройки, аналитика, биллинг)
  - Дублировать роль
  - Удалить роль (проверяется количество сотрудников с этой ролью)
- **Стандартные роли:** Администратор, Менеджер зала, Кассир, Повар, Хостес, Контент-менеджер
- **Пустое состояние:** «Нет ролей»

### Филиалы
- **URL:** /team/branches
- **Действия:**
  - Кнопка «Добавить» → форма создания филиала
  - Список: активные и архивные филиалы
  - Кнопка «Восстановить» для архивных
  - Тег «Неактивен» для неактивных филиалов
  - Предупреждение «Нет зон доставки — доставка не работает» → «Настроить зоны»
  - Ограничение по тарифу: «На вашем тарифе доступно {N} филиалов»
- **Пустое состояние:** «Нет активных филиалов.»
