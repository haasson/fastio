# help — заметка для агента

Помощь, туры, ссылки на базу знаний (shared). Полная мета — `feature.manifest.ts`.

## Что модуль делает

Пошаговые туры по UI (driver.js-like) + ссылки в публичную базу знаний (`apps/help`). Туры описываются декларативно в `tours/<id>.ts`, прогресс хранится в `tenants.tours_state`.

## Карта модуля

| Файл | Что внутри |
|---|---|
| `composables/useTour.ts` | Запуск/пауза/завершение тура + персистенс прогресса |
| `tours/*` | **Внутренние модули туров.** Импорт deep-path `~/features/help/tours/<id>` |

## Типовые задачи

- **Новый тур:** новый файл `tours/<id>.ts` с массивом шагов (`{ target: '#selector', title, content, placement }`) + при необходимости триггер запуска в `useTour.startIf<Condition>`.
- **Ссылка из тура в KB:** статья — в `packages/kb/content/<slug>.md`, в шаге укажи `kbSlug` и `useTour` сам отрендерит deep-link на `apps/help`.
- **Контекстная подсказка на странице:** не тур, а tooltip — клади в `apps/admin/shared/ui/components/UiHint.vue`, не сюда.

## Антипаттерны (не делай так)

- ❌ Хранить тексты туров в БД — туры это код (`tours/*.ts`), потому что они привязаны к селекторам UI, который тоже код.
- ❌ Дублировать KB-статьи в шагах тура — давай ссылку.
- ❌ Запускать тур из layout автоматически без проверки `tenants.tours_state` — будут раздражать.

## Куда расти

Видео-туториалы — в KB (`@fastio/kb`), не сюда. Тур указывает ссылку.
