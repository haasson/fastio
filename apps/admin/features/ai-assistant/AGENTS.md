# ai-assistant — заметка для агента

AI-ассистент админки (shared). Полная мета — `feature.manifest.ts`.

## Что модуль делает

Чат-виджет в админке: пользователь спрашивает «как сделать X», ассистент ищет в `@fastio/kb` и отвечает + при необходимости даёт deep-link на нужный раздел.

**Серверная часть живёт в `apps/admin/server/ai/*`** (Nitro server endpoints) — это not in modules: AI-routes специально не вертикальные, и они работают с секретами провайдера. По плану `server/` отдельная вселенная — ESLint правила вертикалей на неё не действуют.

## Карта модуля

| Файл | Что внутри |
|---|---|
| `composables/useAiChat.ts` | Состояние чата + стриминг ответа с `/api/ai/chat` |
| `components/AiChat.vue` | Виджет чата. Импорт deep-path `~/features/ai-assistant/components/AiChat.vue` |
| `server/ai/*` (НЕ в features) | Серверные endpoints: чат, RAG поверх KB, контекст тенанта |

## Типовые задачи

- **Новая команда/инструмент для ассистента:** в `server/ai/tools/<name>.ts` (Tool calling). KB-контекст и сам промпт — там же.
- **Расширить контекст тенанта:** в `server/ai/context.ts` — собирается сводка тенанта (план, модули) для системного промпта.
- **Обновить базу знаний:** правь `.md` в `packages/kb/content/*`. Хук на коммит должен прогнать обновление; см. CLAUDE.md правило про KB актуальность.

## Антипаттерны (не делай так)

- ❌ Делать вызовы LLM с фронта — провайдер-ключ только на сервере (`server/ai/*`).
- ❌ Хранить историю чата в БД без явного запроса — приватность. Хранится клиентски.
- ❌ Дёргать `@fastio/kb` напрямую из `useAiChat` — KB на сервере (RAG), фронт получает только финальный ответ.
- ❌ Дублировать «как-то знать» структуру модулей в системном промпте — серверный контекст уже знает все `feature.manifest.ts`. (Идея на расти: автоматически инжектить summary всех манифестов в системный промпт ассистента.)
