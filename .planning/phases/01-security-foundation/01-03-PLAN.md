<objective>
Lock down the last public table without RLS (edge_alerts_state) and add a CI audit step that blocks any future migration from regressing TENANT_TABLES protection.

Purpose: SEC-01 establishes the "tenant isolation is airtight" half of Phase 1. The live-DB audit in RESEARCH.md confirmed all 46 TENANT_TABLES already have rowsecurity=true — the actual remediation work is one table (edge_alerts_state) plus a CI gate to prevent regression. Without the CI gate, a future migration could ALTER TABLE ... DISABLE ROW LEVEL SECURITY (intentional or accidental) and leak a tenant's data through anon-role reads.

Output: a new migration 302_fix_edge_alerts_rls.sql adding RLS to edge_alerts_state (no policies = default-deny; pg_cron continues to work via SECURITY DEFINER bypass); a new audit step appended to the migrate.yml Apply pending migrations run block that queries pg_class after migrations apply and fails the workflow if any TENANT_TABLE has rowsecurity=false.
</objective>
