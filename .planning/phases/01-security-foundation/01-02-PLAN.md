<objective>
Add a CI guard that fails the build if the Supabase service-role key ever leaks into the storefront public bundle.

Purpose: SEC-02 closes the highest-impact key-leak vector. Nuxt 3 runtimeConfig correctly scopes supabaseServiceRoleKey as server-only today, but there is no automated check ensuring this contract holds across refactors. A single misplaced runtimeConfig.public assignment or a stray inline reference in client code would silently ship the service-role key to every storefront visitor — a catastrophic, irrecoverable exposure. This CI job converts that silent failure mode into a hard build break.

Output: a new security job in .github/workflows/ci.yml that runs after the existing check job, only on push to main (per D-04 — PRs don't pay the storefront-build cost), builds the storefront, and greps .output/public/ for the literal string service_role. A match exits non-zero with a ::error:: annotation. No match prints "Security check passed".
</objective>
