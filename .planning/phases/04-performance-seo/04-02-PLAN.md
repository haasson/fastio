<objective>
Install `@nuxt/image` with the IPX provider, configure Nitro `routeRules` with `Vary: Host` for tenant-isolated CDN caching, and replace four `<img>` tags / one CSS-background image on critical above-fold and below-fold storefront components with `<NuxtImg>` carrying explicit `width`/`height`/`format="webp"` and correct `loading`/`fetchpriority` per the LCP decision matrix. Closes PERF-03 and PERF-04.

Purpose: The storefront serves raw originals (often multi-MB JPGs from Supabase Storage) without dimensions — that's the dominant LCP and CLS regression. CSS `background-image` in the hero cannot get `fetchpriority="high"`, so the LCP candidate is browser-deprioritized. Without `Vary: Host`, any reverse-proxy/CDN between Traefik and the browser will mix cached HTML across tenants — a silent multi-tenant data leak waiting to happen.

Output: WebP-optimized, dimension-correct image rendering across logo + hero + banners + product cards; tenant-isolated SSR caching via response headers; IPX domain whitelist restricted to the project's Supabase Storage subdomain to block SSRF.
</objective>

## Phase Goal slice

**As a** storefront visitor in CIS on mobile, **I want to** open a venue page quickly, **so that** I don't bounce on a slow load.

This plan delivers the "page loads fast" slice: after it ships, a real user opens a storefront on a mid-tier Android device and sees the hero + logo render as WebP with no layout shift, while the CDN/reverse-proxy never mixes tenant-A's cached HTML into a tenant-B request.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External Supabase Storage URL → IPX → response stream | IPX proxies arbitrary URLs from its `domains` whitelist; an open whitelist is an SSRF gateway into the Nitro container's network. |
| Storefront SSR HTML cache → Traefik/CDN → other tenants | Without `Vary: Host`, downstream caches will serve tenant-A's HTML to tenant-B requests on the same edge node. |
| IPX `/_ipx/**` route → user supplied URL parameter | Attackers may craft URLs to probe internal services if the domains whitelist is wildcard or missing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-04 | Information disclosure | IPX `domains` config in nuxt.config.ts | mitigate | Whitelist is `[process.env.NUXT_PUBLIC_SUPABASE_URL?.replace('https://', '') ?? '']` — a single specific subdomain. Explicitly NOT `*.supabase.co` (no wildcard). Task 1 acceptance asserts the literal absence of the wildcard. |
| T-4-05 | Information disclosure | Multi-tenant SSR HTML cache | mitigate | `routeRules['/**'].headers.vary = 'Host'` instructs every downstream cache layer (Traefik, optional CDN) to key responses by Host header. Tenant-A's cached HTML can never be served to tenant-B requests. |
| T-4-06 | Information disclosure | /api/** responses | mitigate | `routeRules['/api/**'].headers['cache-control'] = 'no-store'` — tenant data API responses never cached by any intermediary. |
| T-4-SC | Tampering | npm install of `@nuxt/image@2.0.0` | mitigate | RESEARCH.md Package Legitimacy Audit verified: official Nuxt module, github.com/nuxt/image, ~4.5y old, high downloads, mainstream maintainer (Approved). slopcheck not run (unavailable in research); manual verification documented. Pinned to exact version 2.0.0 in package.json install command. |
