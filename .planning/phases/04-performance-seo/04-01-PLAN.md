<objective>
Fix OG/SEO metadata in the storefront root component so that pasting a storefront URL into Telegram, WhatsApp, or any OG-aware scraper produces a meaningful preview card showing the venue name, description, and logo image. Closes PERF-01.

Purpose: Social previews are the first impression for a shared storefront link. Today the `useHead` block in `app.vue` is missing `og:url`, may emit a relative `og:image` (which Telegram/WhatsApp silently drop), and has no Twitter card or canonical link. Scrapers do not execute JavaScript and never follow redirects, so every OG tag MUST be present in the SSR HTML with an absolute URL.

Output: Updated `useHead` block in `apps/storefront/app.vue` that emits a complete OG + Twitter + canonical meta set sourced from tenant data, plus a Vitest unit test verifying the head-config object shape.
</objective>

**As a** storefront visitor in CIS on mobile, **I want to** open a venue page quickly and see meaningful previews when sharing the link, **so that** I can decide to order without bouncing on a slow load or a blank preview card.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tenant DB → SSR HTML | `tenant.seo.ogImage` and `tenant.siteContent.logo` are user-supplied URLs stored in Supabase; they cross into raw HTML via meta tag `content` attribute. |
| Storefront SSR → Social scraper (Telegram/WhatsApp/Facebook) | Outbound — OG tags are read by third-party scrapers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Tampering | `buildHead` og:image attribute | mitigate | Guard `rawOgImage.startsWith('http')` before inserting into HTML head. Rejects `javascript:`, `data:`, and relative URLs that scrapers either silently drop or that could enable XSS via meta-content injection. Test 1 in `app.test.ts` asserts this. |
| T-4-02 | Information disclosure | og:url from useRequestURL() | mitigate | Use `tenant.customDomain ?? slug + '.fastio.ru'` instead of `useRequestURL()` — prevents leaking internal Coolify hostnames (Pitfall 2 in RESEARCH.md). Null-safe form prevents the dual regression of emitting `https://undefined.fastio.ru` when tenant is null (Test 7 locks this). |
| T-4-03 | Spoofing | Tenant-controlled og:title/description | accept | Tenants control their own venue's preview content — by design. Owner admin auth gates the seo.* mutations elsewhere. |
</threat_model>
