<objective>
Codify REL-01 as a repeatable, CI-enforced audit: every `supabase.channel()` call must be paired with a `removeChannel()` so realtime subscriptions never leak. After this plan, the audit returns zero violations on the current codebase (phase success criterion 3) and a future leak fails CI.

Purpose: Leaked realtime channels are an "invisible failure" — they accumulate silently and surface as noise once E2E (Phase 3) runs. STATE.md placed REL-01 here deliberately. The D-09 grep formula from CONTEXT.md is brittle (it whitelists by util filename and excludes only .output/.nuxt, so it falsely reports node_modules cache hits and the legitimately-cleaned useTableRealtime / backoffice SSE files). This plan ships a corrected co-presence formula: every file containing `.channel(` must also contain `removeChannel`, excluding node_modules and build dirs. Verified to return clean on the current tree.
Output: An audit script and a CI step that runs it.
</objective>
