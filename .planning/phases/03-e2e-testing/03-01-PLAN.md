<objective>
Поднять изолированный staging Supabase Cloud проект, подключить его к pipeline миграций, создать минимальный E2E seed и переписать Playwright globalSetup так, чтобы он мог работать против staging через service-role HTTP API (без зависимости от docker), сохранив существующий локальный docker-путь без изменений.

Назначение: SEC-04 закрывает гэп «E2E тесты не должны делить БД с чем-либо ещё». Сейчас E2E-сьют гоняется против локального Supabase через docker. Чтобы фаза 3 (E2E flows) могла надёжно работать в CI без зависимости от ноутбука разработчика, надо иметь staging-окружение и dual-path globalSetup. Перенастройка `e2e-smoke.yml`/`e2e-nightly.yml` под staging происходит в последующих планах (03-02..03-05) и финализируется в общем suite-проходе — этот план поставляет staging-проект, push-миграций, seed-файл и dual-path globalSetup.
