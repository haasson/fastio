# Testing Patterns

**Analysis Date:** 2026-05-20

## Test Framework

**Unit/Integration Runner:**
- Vitest 4.x
- Config: `/vitest.config.ts` (root, covers all workspaces)
- Environment: `happy-dom`
- Globals: `true` (but imports still explicit in test files)

**Assertion Library:**
- Vitest built-in (`expect`)

**E2E Runner:**
- Playwright 1.60+
- Config: `/playwright.config.ts`
- Browser: Chromium only (Desktop Chrome)
- Workers: 1 (shared DB state — no parallelism)
- Retries: 1 in CI, 0 locally

**Run Commands:**
```bash
pnpm test              # Vitest watch mode
pnpm test:run          # Vitest single run (all unit/integration)
pnpm test:coverage     # Vitest with v8 coverage
pnpm vitest run <path> # Single test file
pnpm test:e2e          # Playwright E2E (requires running dev servers)
pnpm test:e2e:ui       # Playwright interactive UI mode
pnpm test:e2e:setup    # Generate .fixtures.json from DB
```

## Test File Organization

**Location:**
- Unit/integration tests: co-located in `__tests__/` subdirectory within the module
- E2E tests: separate `/tests/e2e/` directory at repo root

**Naming:**
- Unit test files: `<subject>.test.ts` (e.g., `orders.test.ts`, `order-events.test.ts`, `cart.test.ts`)
- E2E test files: `<flow>.spec.ts` (e.g., `order-flow.spec.ts`, `admin-login.spec.ts`)
- ESLint barrier tests: `eslint-barrier.test.ts`

**Structure:**
```
apps/admin/
  __tests__/
    eslint-barrier.test.ts          # Architecture enforcement test
  config/__tests__/
    team-roles.test.ts
  features/<feature>/__tests__/
    <subject>.test.ts
  test-utils/
    nuxt-imports-stub.ts            # Stub for Nuxt virtual #imports
    vue-router-stub.ts              # Stub for vue-router

apps/storefront/
  features/<feature>/__tests__/
    <subject>.test.ts
  server/__tests__/
    cross-tenant.test.ts
    bulk.post.test.ts

packages/shared/src/__tests__/
  <utility>.test.ts                 # Pure utility/helper tests

tests/e2e/
  *.spec.ts                         # Playwright E2E flows
  fixtures.ts                       # Type definition + loader for .fixtures.json
  .fixtures.json                    # Generated by setup script (gitignored)
  global-setup.mjs                  # Runs before all E2E tests
```

## Test Structure

**Suite Organization:**
```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest'

// Factory helper — always use overrides pattern
const makeSomething = (overrides: Partial<Type> = {}): Type => ({
  id: 'default-id',
  field: 'default-value',
  ...overrides,
})

describe('SubjectUnderTest', () => {
  beforeEach(() => {
    vi.clearAllMocks()
    // reset reactive refs
  })

  describe('methodName()', () => {
    it('описание на русском — конкретное условие → ожидаемый результат', () => {
      const result = methodName(makeSomething())
      expect(result.field).toBe('expected')
    })
  })
})
```

**Patterns:**
- `beforeEach` for resetting mocks and state: `vi.clearAllMocks()`, reset reactive `ref` values
- `afterEach` for cleanup (e.g., `localStorage.clear()`)
- Test descriptions in Russian matching domain language
- Import subject after `vi.mock()` calls for composables with module-level mocks:
  ```typescript
  vi.mock('~/shared/stores/tenant', () => ({ ... }))
  // Import after mocks
  import { useOnboarding } from '../composables/useOnboarding'
  ```

## Mocking

**Framework:** Vitest `vi.mock()`, `vi.fn()`, `vi.spyOn()`

**Module mocking patterns:**
```typescript
// Pinia store mock
vi.mock('~/shared/stores/tenant', () => ({
  useTenantStore: () => ({
    maybeTenant: tenantRef,
    isOwner: isOwnerRef,
    update: updateMock,
  }),
}))

// Feature composable mock (gate)
vi.mock('~/shared/plan/useGate', () => ({
  useGate: () => ({
    delivery: computed(() => ({ enabled: gateDelivery.value, reason: null })),
  }),
}))

// Observability mock (to assert reportError calls)
vi.mock('@fastio/shared/observability', () => ({ reportError: vi.fn() }))

// pinia storeToRefs mock
vi.mock('pinia', () => ({
  storeToRefs: (store: { field: unknown }) => ({ field: store.field }),
}))
```

**What to Mock:**
- Pinia stores (`~/shared/stores/*`) when testing composables that depend on them
- Feature gates (`~/shared/plan/useGate`) using reactive `ref` booleans for gate values
- `@fastio/shared/observability` to assert `reportError` is called on errors
- Nuxt virtual modules (`#imports`) via `apps/admin/test-utils/nuxt-imports-stub.ts` (aliased in `vitest.config.ts`)
- `vue-router` via `apps/admin/test-utils/vue-router-stub.ts`
- Supabase client for server endpoint tests (fluent chain mock)

**What NOT to Mock:**
- `@fastio/shared` utilities — test against the real implementation
- Pure mapping functions (`mapOrder`, `mapOrderEvent`) — no mocks needed
- `vue` / `pinia` core unless specifically testing reactive behavior in isolation
- `localStorage` — use real `happy-dom` localStorage, clear in `afterEach`

**Pinia in tests:**
```typescript
import { setActivePinia, createPinia } from 'pinia'

beforeEach(() => {
  setActivePinia(createPinia())
  store = useCartStore()
})
afterEach(() => {
  localStorage.clear()
})
```

## Fixtures and Factories

**Factory helpers (all test files use this pattern):**
```typescript
const makeOrderRow = (overrides: Record<string, unknown> = {}): Record<string, unknown> => ({
  id: 'order-1',
  tenant_id: 'tenant-1',
  customer_name: 'Иван',
  status: 'new',
  // ... all required fields with sensible defaults
  ...overrides,
})

const makeDish = (overrides: Partial<DishCartItem> = {}): DishCartItem => ({
  kind: 'dish',
  _key: 'k1',
  dishId: 'd1',
  price: 500,
  quantity: 1,
  // ...
  ...overrides,
})
```

**Typed factories:** Always type the return value explicitly. Use `Partial<T>` for overrides.

**E2E Fixtures:**
- Generated by `pnpm test:e2e:setup` → `/tests/e2e/.fixtures.json` (gitignored)
- Typed in `/tests/e2e/fixtures.ts`
- Contains: `phoneMarker`, `retailTenantSlug`, `servicesTenantSlug`, `adminEmail`, `adminPassword`, session tokens
- Loaded synchronously via `readFileSync` (Playwright CJS bundle constraint)
- Helper `tgSessionCookie(vertical)` for authenticated E2E sessions

## Test Types

**Unit Tests (pure functions):**
- Scope: Individual mapper functions, utility functions, pure business logic
- Location: `packages/shared/src/__tests__/`, `features/<X>/__tests__/`
- Example subjects: `mapOrder`, `mapOrderEvent`, `getItemUnitPrice`, `formatEventText`, `pluralize`
- No mocks needed — test inputs → expected outputs directly

**Integration Tests (composables with state):**
- Scope: Composables, Pinia stores, composable behavior under different conditions
- Location: `features/<X>/__tests__/`
- Uses: `setActivePinia(createPinia())`, `vi.mock()` for dependencies, reactive `ref` for controllable inputs
- Example subjects: `useOnboarding`, `useCartStore`, `useKitchenStatusBlock`

**Architecture/Barrier Tests:**
- Scope: ESLint vertical isolation rules
- Location: `apps/admin/__tests__/eslint-barrier.test.ts`
- Uses: `ESLint` API directly to lint synthetic code snippets and assert rule violations
- Verifies: services cannot import retail, retail cannot import services, shared cannot import verticals, cross-module deep imports blocked

**Server Endpoint Tests:**
- Scope: Nitro server API handlers for security (cross-tenant isolation)
- Location: `apps/storefront/server/__tests__/`
- Uses: Fluent Supabase chain mock, h3 utilities mocked
- Verifies: Every endpoint applies `.eq('tenant_id', X)` filter

**E2E Tests:**
- Scope: Full happy-path user flows (storefront + admin)
- Location: `/tests/e2e/`
- Flows covered: `order-flow`, `reservation-flow`, `appointment-flow`, `admin-login`, `admin-create-dish`, `account-auth`, `smoke-admin`, `smoke-storefront`
- Each spec overrides `baseURL` via `test.use({ baseURL: ... })` for multi-tenant routing
- `data-testid` attributes used for element selectors

## Common Patterns

**Async Testing (composable methods):**
```typescript
it('advances currentStepId to the next step', async () => {
  tenantRef.value = makeTenant()
  await useOnboarding().completeStep('category')
  expect(updateMock).toHaveBeenCalledTimes(1)
  expect(updateMock.mock.calls[0][0].onboardingState.currentStepId).toBe('item')
})
```

**Error Testing (reportError assertion):**
```typescript
it('broken JSON в localStorage — не падает + reportError', async () => {
  const { reportError } = await import('@fastio/shared/observability')
  localStorage.setItem('cart', '{{invalid-json')

  const store2 = useCartStore()
  store2.restore()

  expect(store2.items).toHaveLength(0)
  expect(reportError).toHaveBeenCalled()
})
```

**Reactive state control in tests:**
```typescript
// Top-level reactive refs control mock behavior
const gateDelivery = ref(false)

vi.mock('~/shared/plan/useGate', () => ({
  useGate: () => ({
    delivery: computed(() => ({ enabled: gateDelivery.value, reason: null })),
  }),
}))

// Flip in individual tests
gateDelivery.value = true
const ids = useOnboarding().steps.value.map((s) => s.id)
expect(ids).toContain('intake-delivery')
```

**localStorage testing (cart persistence):**
```typescript
it('persist writes to cart:<slug>, not legacy cart', () => {
  store.add(makeDish({ dishId: 'd-key' }))
  const tenantScoped = localStorage.getItem('cart:localhost')
  const legacy = localStorage.getItem('cart')
  expect(tenantScoped).not.toBeNull()
  expect(legacy).toBeNull()
})
```

**E2E data-testid selectors:**
```typescript
// Prefer data-testid over CSS selectors
await page.getByTestId('cart-fab').click()
await page.getByTestId('checkout-phone').fill(fixtures.phoneMarker)

// :visible modifier for components with mobile/desktop dual DOM
const card = page.locator('[data-testid="product-card-dish"]:visible', { hasText: 'Маргарита' }).first()
await card.waitFor({ state: 'visible', timeout: 15_000 })
```

## Vitest Aliases (Test-only Stubs)

The root `vitest.config.ts` sets up aliases to replace Nuxt/vue-router virtual modules:
- `#imports` → `apps/admin/test-utils/nuxt-imports-stub.ts` — stubs `useNuxtApp`, `useRuntimeConfig`, `navigateTo`, `defineNuxtPlugin`
- `vue-router` → `apps/admin/test-utils/vue-router-stub.ts` — stubs `onBeforeRouteLeave`
- `h3` → hardcoded pnpm store path (tech debt — see `TECHDEBT.md` and comment in `vitest.config.ts`)

Test suites can override these with `vi.mock()` when they need specific behavior.
