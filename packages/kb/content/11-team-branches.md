# Филиалы и заведение

Раздел в левом меню называется **«Филиалы»** (если тариф включает мультифилиальность) или **«Заведение»** (если у тенанта одна точка).

## Single-branch — режим одного заведения (venue mode)

Если тариф не включает модуль мультифилиалов — раздел называется **«Заведение»**. На странице сразу видны две формы:

1. **Адрес и координаты** — адрес единственной точки с подсказками (DaData), мини-карта Яндекс.Карт с маркером, координаты подтягиваются автоматически после выбора адреса
2. **Общие настройки** — телефон заведения и часы работы (хранятся на уровне тенанта)

Сверху страницы — апселл-предложение перейти на тариф Pro для добавления нескольких точек.

В этом режиме нет списка филиалов, кнопок «Добавить»/«Архивировать» и drawer-редактора.

## Опт-аут филиалов на онбординге

Если на шаге «Первый филиал» в онбординге был отмечен чекбокс «Не указывать филиал» (для выездных услуг или удалённой работы) — раздел «Филиалы» в админке скрывается полностью. Вернуть доступ можно только через поддержку.

## Multi-branch — несколько точек

Если у вас несколько точек — каждая настраивается как отдельный филиал. Заказы, зоны доставки и статистика разделяются по филиалам.

Структура страницы:

1. **Сверху** — блок «Общие настройки заведения» (телефон тенанта + базовое расписание). Эти значения **наследуются** всеми филиалами по умолчанию. У каждого филиала можно переопределить.
2. **Ниже** — список активных филиалов с кнопкой «Добавить» (если не достигнут лимит тарифа), а также архив (если есть архивные).

> Количество филиалов ограничено тарифом. При достижении лимита кнопка «Добавить» становится неактивной.

## Создание/редактирование филиала (multi-branch)

Кнопка «**+ Добавить**» вверху списка или иконка карандаша на строке филиала открывают drawer.

### Обязательные поля

**Название** *(обязательно)* — «Центральный», «Левый берег», «Аэропорт».

**Цвет филиала** — выберите из палитры. Цвет помогает визуально различать точки в интерфейсе.

### Адрес и координаты

**Адрес** — введите адрес с подсказками. После выбора варианта из выпадающего списка — на карте ниже появится маркер. Координаты подтягиваются автоматически.

### Дополнительные поля (только при нескольких филиалах)

| Поле | Описание |
|------|----------|
| **Телефон** | Контактный телефон филиала. Если не задан — используется телефон из «Общие настройки» |
| **Своё расписание** | Тогл, переключающий филиал на индивидуальные часы работы вместо унаследованных |
| **Префикс номера заказа** | Например «MSK» → заказы будут «MSK-001». Доступно только при включённой нумерации «по филиалам» в настройках |
| **Филиал активен** | Выключенный филиал скрыт с витрины, но остаётся в админке |

> Per-branch Telegram/email уведомления не поддерживаются — уведомления настраиваются глобально на тенанте в **Настройки → Уведомления**.

## Управление доступом по филиалам

При приглашении сотрудника в **Команда → Участники** можно ограничить его видимость конкретными филиалами.

Сотрудник без явных ограничений видит все филиалы.

## Зоны доставки

Каждый филиал имеет свои зоны доставки. Настраиваются в **Заказы → Доставка**.

> ⚠️ Если у филиала нет зон доставки — доставка для него не работает. Система покажет предупреждение прямо в списке филиалов.

## Архивирование

Кнопка «Архивировать» убирает филиал из активных. Данные сохраняются.

**Нельзя архивировать** если есть активные заказы — сначала завершите или отмените их.

Кнопка **скрыта**, если у тенанта остался только один филиал — тогда нечего архивировать.

Восстановить из архива можно в любой момент (если позволяет лимит тарифа).

## Переключение между филиалами

В левом нижнем углу навигации — переключатель текущего филиала. Владелец видит «Все филиалы» и может смотреть сводную статистику.

Сотрудник с доступом только к отдельным филиалам видит только их.
