# Управление столами

> Требует включения модуля **Столы** в Настройки → Модули

Раздел для работы с обслуживанием в зале: открытие столов, добавление блюд, отслеживание готовности, расчёт с гостями.

## Модель: стол = один редактируемый чек

Открытый стол держит **один чек** — растущий заказ, в который и официант, и гость по QR дозаписывают блюда в течение всей посадки. Расчёт — один раз в конце: скидка, способ оплаты, закрытие стола. После расчёта чек попадает в историю как одна оплаченная посадка.

## Страница столов

Разделена на две секции:

**Открытые столы** — столы с активным чеком. На каждой карточке видно: текущую сумму, количество вызовов, блюда на кухне, готовые блюда.

**Закрытые столы** — свободные столы. Кнопка «Открыть стол» создаёт новый пустой чек.

## Столы и филиалы

Каждый стол принадлежит конкретному филиалу. Привязка нужна, чтобы dine-in заказ со стола (через QR) поступал на кухню и в нумерацию именно того филиала, где стоит стол.

- **Один филиал** — всё как обычно, выбор филиала не требуется, стол создаётся в этом филиале.
- **Несколько филиалов + выбран конкретный** (переключатель филиала в боковом меню) — на странице видны только столы выбранного филиала; новый стол создаётся в нём.
- **Несколько филиалов + «Все филиалы»** — открывается **сводка только для чтения**: по каждому филиалу видно количество столов (всего / занято / свободно). Создавать и редактировать столы здесь нельзя — выберите конкретный филиал в боковом меню.

Стол нельзя оставить без филиала. Поэтому филиал со столами **нельзя архивировать** — сначала перенесите или удалите его столы (см. «Команда → Филиалы»).

## Создание стола

На странице **Столы → Список** кнопка «+ Добавить стол». Стол создаётся в филиале, выбранном в боковом меню (при единственном филиале — в нём).

| Поле | Обязательно | Описание |
|------|-------------|----------|
| **Название** | Да | Например: «Стол 1», «Веранда 3», «VIP» |
| **Вместимость** | Нет | Количество мест от 1 до 100 |
| **Форма** | — | «Прямоугольный» или «Круглый» — для схемы зала |
| **Заметки** | Нет | Дополнительная информация |
| **Активен** | — | Тогл. Неактивный стол не отображается |

## Открытие стола

Нажмите «Открыть стол» на закрытом столе — создаётся пустой чек. Теперь к столу можно добавлять блюда. Если на столе была активная бронь — она привязывается к чеку.

## Добавление блюд

Кнопка «+ Блюдо» на карточке открытого стола.

Откроется пикер блюд из меню. Выберите нужные позиции, укажите количество. Если у блюда есть модификаторы — их тоже нужно выбрать.

Блюда дозаписываются в чек стола. Добавленные официантом позиции сразу уходят в кухонную очередь (если модуль кухни включён). Позиции, добавленные гостем по QR, ждут подтверждения персоналом и уходят на кухню только после него.

## Статусы позиций

Каждая позиция в чеке проходит через статусы:
- **Ожидает** — добавлено, но не подтверждено официантом
- Обычный — позиция принята

Позиции в статусе «Ожидает» видны с оранжевой меткой. Подтвердите кнопкой ✓ или отклоните ✗.

**Подтвердить все** — кнопка которая разом подтверждает все ожидающие позиции.

## Что показывает карточка стола

Сверху — время открытия («Открыт 14 мин назад»), текущая сумма чека и активные вызовы гостей (просроченные подсвечены красным).

Позиции разнесены по этапам; блок виден всегда, пока в нём что-то есть:
- **Заказ со стола** — только что добавленные позиции, ждут подтверждения официантом. Подтвердить по одной (✓) или разом — **«Подтвердить все»**.
- **Готовка на кухне** — позиции в работе (в очереди / готовятся).
- **Готовые — забрать** — приготовленные блюда. «Забрал» по одной или сразу все — **«Забрал все»**.
- **Чек** — уже поданные блюда и итоговая сумма стола. Длинный чек сворачивается: показывается заданное число строк, остальные — под «ещё».

Настройки отображения — **Столы → Настройки**: ширина карточек (компактные / средние / крупные), число строк чека на карточке, показ категории блюда у позиций.

## Что гость может по QR-коду

QR-код стола даёт гостю два интерактива, и каждый включается/выключается отдельно в **Столы → Настройки → «Режим стола (QR)»**:
- **Заказ со стола** — гость сам дозаписывает блюда в чек стола. Его позиции приходят со статусом «Ожидает» и уходят на кухню только после подтверждения персоналом. Гостю по QR промокоды не предлагаются — скидки применяет только персонал при расчёте.
- **Вызов официанта** — кнопки вызовов (см. «Вызовы»).

Комбинации любые: только витрина (оба выключены — гость видит меню без действий), только вызов, только заказ или всё вместе. По умолчанию включено всё. Запреты действуют и на сервере — выключенный заказ/вызов нельзя отправить в обход интерфейса.

## Расчёт (закрытие стола)

Кнопка «Расчёт» открывает модальное окно. Расчёт проводится один раз и закрывает весь чек.

**Что показывает:**
- Полный список позиций чека с ценами
- Блюда на кухне (если ещё есть)

**Скидка:**
- Введите процент (%) или фиксированную сумму (₽)
- Итоговая сумма пересчитывается автоматически

**Способ оплаты** — наличные или карта. Записывается в чек вместе с тем, кто и когда провёл расчёт.

**Закрыть стол** — финализирует чек: фиксирует скидку, оплату, освобождает стол и завершает привязанную бронь. Чек уходит в историю как оплаченная посадка. Пустой чек (без позиций) при закрытии отменяется, а не оплачивается.

Кнопка расчёта неактивна, пока есть неподтверждённые позиции (от гостя по QR) или незавершённые блюда на кухне — сначала подтвердите/отклоните позиции и дождитесь готовности или отмените кухонные блюда.

## Действия со столом

| Кнопка | Что делает |
|--------|-----------|
| **+ Блюдо** | Открывает пикер блюд |
| **Расчёт** | Открывает форму закрытия стола |
| **QR-код** | Показывает QR для самостоятельного заказа гостем |
| **Подтвердить** | Подтверждает одну позицию |
| **Подтвердить все** | Подтверждает все ожидающие позиции |
| **✓ Забрал** | Подтверждает что готовое блюдо выдано |
| **✓ Забрал все** | Отмечает все готовые блюда стола выданными разом |
| **Редактировать** | Настройки стола (название, вместимость) |
| **Удалить блюдо** | Убирает конкретную позицию из чека. Спрашивает подтверждение — действие необратимо, позиция удаляется физически. Если удалена последняя позиция — чек остаётся открытым (пустым), стол не закрывается |

## Советы

- **QR-код на столе** — распечатайте и поставьте на каждый стол. Гости смогут сами дозаписывать блюда в чек стола не ожидая официанта (их позиции вы подтверждаете перед отправкой на кухню).
- **Не закрывайте стол с блюдами на кухне** — дождитесь готовности. Система не даст рассчитать пока есть активные кухонные позиции или неподтверждённые позиции гостя.
- **Скидка при расчёте** — удобна для применения в особых случаях (постоянный гость, ошибка с заказом).
