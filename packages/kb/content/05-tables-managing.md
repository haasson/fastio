# Управление столами

> Требует включения модуля **Столы** в Настройки → Модули

Раздел для работы с обслуживанием в зале: открытие столов, добавление блюд, отслеживание готовности, расчёт с гостями.

## Страница столов

Разделена на две секции:

**Открытые столы** — столы с активной сессией. На каждой карточке видно: текущую сумму, количество вызовов, блюда на кухне, готовые блюда.

**Закрытые столы** — свободные столы. Кнопка «Открыть стол» начинает новую сессию.

## Создание стола

На странице **Столы → Список** кнопка «+ Добавить стол».

| Поле | Обязательно | Описание |
|------|-------------|----------|
| **Название** | Да | Например: «Стол 1», «Веранда 3», «VIP» |
| **Вместимость** | Нет | Количество мест от 1 до 100 |
| **Форма** | — | «Прямоугольный» или «Круглый» — для схемы зала |
| **Заметки** | Нет | Дополнительная информация |
| **Активен** | — | Тогл. Неактивный стол не отображается |

## Открытие сессии

Нажмите «Открыть стол» на закрытом столе — начинается новая сессия. Теперь к столу можно добавлять блюда.

## Добавление блюд

Кнопка «+ Блюдо» на карточке открытого стола.

Откроется пикер блюд из меню. Выберите нужные позиции, укажите количество. Если у блюда есть модификаторы — их тоже нужно выбрать.

Добавленные блюда сразу уходят в кухонную очередь (если модуль кухни включён).

## Статусы позиций

Каждая позиция в сессии проходит через статусы:
- **Ожидает** — добавлено, но не подтверждено официантом
- Обычный — позиция принята

Позиции в статусе «Ожидает» видны с оранжевой меткой. Подтвердите кнопкой ✓ или отклоните ✗.

**Подтвердить все** — кнопка которая разом подтверждает все ожидающие позиции.

## Что показывает карточка стола

На открытом столе:
- Время открытия стола («Открыт 14 мин назад»)
- Текущая сумма сессии
- Список позиций (до 3, остальные скрыты)
- Блюда на кухне с их статусом
- Готовые блюда — «Забрал» чтобы подтвердить выдачу
- Активные вызовы гостей

## Расчёт (закрытие стола)

Кнопка «Расчёт» открывает модальное окно.

**Что показывает:**
- Полный список позиций с ценами
- Блюда на кухне (если ещё есть)

**Скидка:**
- Введите процент (%) или фиксированную сумму (₽)
- Итоговая сумма пересчитывается автоматически

**Закрыть стол** — финализирует сессию. Если на кухне ещё есть незавершённые позиции — кнопка будет неактивна. Нужно дождаться готовности или отменить позиции.

## Действия со столом

| Кнопка | Что делает |
|--------|-----------|
| **+ Блюдо** | Открывает пикер блюд |
| **Расчёт** | Открывает форму закрытия стола |
| **QR-код** | Показывает QR для самостоятельного заказа гостем |
| **Подтвердить** | Подтверждает одну позицию |
| **Подтвердить все** | Подтверждает все ожидающие позиции |
| **✓ Забрал** | Подтверждает что готовое блюдо выдано |
| **Редактировать** | Настройки стола (название, вместимость) |
| **Удалить блюдо** | Убирает конкретную позицию из сессии. Спрашивает подтверждение — действие необратимо, позиция удаляется физически (а если она последняя в заказе — удаляется и сам заказ) |

## Советы

- **QR-код на столе** — распечатайте и поставьте на каждый стол. Гости смогут сами добавлять блюда к заказу не ожидая официанта.
- **Не закрывайте стол с блюдами на кухне** — дождитесь готовности. Система не даст закрыть пока есть активные кухонные позиции.
- **Скидка при расчёте** — удобна для применения в особых случаях (постоянный гость, ошибка с заказом).
