# Нумерация заказов

Настройка формата номера заказа. Номер отображается на карточке заказа, в чеке и в уведомлениях клиенту.

## Форматы номера

Выберите один из четырёх форматов:

| Формат | Пример | Когда использовать |
|--------|--------|--------------------|
| **Только счётчик** | `1042` | Простой вариант. Подходит если у вас одно заведение |
| **Префикс + счётчик** | `ORD-1042` | Если хотите брендированный номер |
| **Дата + счётчик** | `2303-042` | Удобно для архива: сразу видно дату заказа |
| **Префикс + дата + счётчик** | `ORD-2303-042` | Максимально информативно |

## Область нумерации

**Сквозная** — единый счётчик для всего заведения. Заказы из всех филиалов идут в одну нумерацию: 1, 2, 3...

**По филиалам** — каждый филиал имеет свой независимый счётчик. Заказы из разных филиалов не пересекаются. В этом режиме **префикс задаётся в настройках каждого филиала** (Команда → Филиалы → редактировать → «Префикс номера заказа»).

## Настройки счётчика

**Нули слева (паддинг)** — ведущие нули в номере счётчика:
- `0` — без нулей: `1042`, `5`, `234`
- `3` — три знака: `042`, `005`, `234`
- `4` — четыре знака: `0042`, `0005`, `1234`

**Начать с** — стартовое значение счётчика. По умолчанию 1. Полезно если переходите с другой системы и хотите продолжить нумерацию.

**Сброс** — когда счётчик сбрасывается:
- **Никогда** — счётчик только растёт
- **Ежедневно** — в полночь счётчик сбрасывается до значения «Начать с»

## Формат даты

Когда формат включает дату, выберите как она записывается в номере:

| Формат | Пример для 23 марта | Применение |
|--------|---------------------|------------|
| `DDMM` | `2303` | Короткий, только день и месяц |
| `DDMMYY` | `230325` | С двузначным годом |
| `YYYYMMDD` | `20250323` | Полная дата, удобна для сортировки |

## Превью

Под настройками показывается пример номера на основе текущих настроек и сегодняшней даты. Меняйте настройки — превью обновляется в реальном времени.

## Советы

- **Ежедневный сброс** удобен если хотите видеть «заказ №5 за сегодня» вместо «заказ №4729 за всё время».
- **Паддинг из 3 нулей** (`042`) выглядит аккуратнее чем просто `42`, особенно в чеках.
- **Нумерация по филиалам** помогает диспетчеру сразу видеть откуда заказ: `MSK-042` из Москвы, `SPB-017` из Питера.
