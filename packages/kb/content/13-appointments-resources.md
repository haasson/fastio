# Сотрудники и объекты

Запись принимают **сотрудники** (мастер, инструктор, тренер) и **объекты** (бильярдный стол, сауна, велосипед, кабинет). У каждого — своё расписание и список оказываемых услуг.

## Сотрудники

**Запись → Сотрудники** → кнопка «Добавить».

| Поле | Описание |
|------|----------|
| **Выбрать из команды** | Переключатель. Включён — выбираете участника из Команды, имя берётся из его аккаунта. Выключен — вводите произвольное имя (для внешних мастеров или сотрудников без доступа в систему). |
| **Филиалы** | В каких филиалах работает. Пусто — во всех. |
| **Услуги и категории** | Дерево с чекбоксами. Можно выбрать категорию целиком (все текущие и будущие услуги в ней) или отдельные услуги внутри — для случаев когда мастер делает только часть услуг из категории. |

Один аккаунт из Команды можно привязать только к одному сотруднику.

## Объекты

**Запись → Объекты** → кнопка «Добавить». Используется для всего, что не привязано к человеку: столы, кабинеты, оборудование, помещения.

| Поле | Описание |
|------|----------|
| **Название** | Например «Бильярдный стол» или «Сауна» |
| **Количество** | Сколько одновременных бронирований допустимо. 10 столов = 10 — клиенты будут записываться на одно и то же время, пока не кончатся свободные. |
| **Филиалы / Категории / Услуги** | Аналогично сотрудникам |

Если стол на ремонте — уменьшите **Количество** на единицу. После починки верните обратно.

## Расписание

Расписание настраивается после создания. Можно:

- **По дням недели** — переключатели и время начала/конца на каждый день
- **Перерывы** — выключенные слоты внутри рабочего дня (обед, технический перерыв)
- **Особые даты** — нестандартные часы на конкретную дату (например, сокращённый день перед праздником). Можно отметить «Повторять ежегодно» — тогда исключение будет срабатывать каждый год в ту же дату
- **Отпуска и отсутствия** — отдельная секция для периодов, когда исполнитель полностью не работает: отпуск, больничный, обучение, другое. Указывается период (С — По) одной строкой и тип. При создании система проверяет нет ли в этом периоде уже забронированных записей — если есть, сначала просит их перенести
- **Применить шаблон** — копирует готовый шаблон («Утро», «2 через 2» и т. п.) из раздела **Графики работы**

Слоты генерируются автоматически из расписания и **шага слота** (настройка в разделе «Настройки записи»).

## Услуги без исполнителя

Если у услуги нет ни одного исполнителя — слоты считаются по графику работы **филиала** (раздел «Заведение»). Это для услуг, где исполнитель не критичен — массаж в одном кресле, аренда снаряжения, посещение бассейна.

## Видимость

Если у услуги один исполнитель — клиент его не выбирает (шаг скрывается). Если **Выбор исполнителя** на услуге выключен — клиент тоже не выбирает, система автоматически распределяет.

## Авто-распределение и количество (capacity)

Когда клиент не указал конкретного исполнителя или объект, система перебирает кандидатов и берёт первого, у которого ещё есть свободное место в этом окне. Для объектов с **Количеством ≥ 2** одно занятое окно не делает ресурс полностью занятым — пока пересечений меньше количества, объект остаётся доступен.

Кандидаты собираются из обоих источников привязки одновременно — выбранных категорий и штучно привязанных услуг.
