# Блюда

Раздел «Меню → Блюда» — основа вашего каталога. Здесь создаются, редактируются и организуются все позиции, которые видит покупатель на витрине.

## Как устроена страница

Слева — список категорий с количеством блюд в каждой. Справа — блюда выбранной категории. Когда вы кликаете на категорию, список справа обновляется.

Тип отображения зависит от типа категории:
- **Обычная категория** → список блюд
- **Комбо-категория** → список комбо-наборов
- **Виртуальная категория** → блюда из других категорий, отфильтрованные по тегу

## Создание блюда

Нажмите «+ Добавить блюдо» при выбранной категории. Откроется форма с несколькими секциями.

### Основная информация

| Поле | Обязательно | Описание |
|------|-------------|----------|
| **Название** | Да | Именно это увидит покупатель. Пример: «Маргарита» |
| **Цена, ₽** | Да | Целое число или дробное. Минимум — 0 |
| **Описание** | Нет | Короткое описание, показывается под названием |
| **Подробное описание** | Нет | Включается отдельным чекбоксом. Показывается при открытии карточки блюда |
| **Фото** | Нет | Соотношение сторон 1:1. Рекомендуется квадратное фото |
| **Категория** | Нет | Показывается, только если категорий больше одной |
| **Вес** | Нет | Число + единица «г» или «мл» |

> **Совет.** Заполните описание и фото — блюда с фото продаются лучше. Вес указывайте для блюд, где это важно покупателю (супы, горячее).

### Ингредиенты

Список ингредиентов, которые покупатель может исключить при заказе. Например: «Лук», «Оливки», «Острый соус».

Добавление: кнопка «+ Добавить ингредиент» → введите название → Enter или следующий клик. Пустые строки игнорируются при сохранении.

Покупатель увидит на витрине чекбоксы «убрать лук», «убрать оливки» и т.д.

> **Копирование.** Если у другого блюда уже есть аналогичный набор ингредиентов — используйте кнопку «Скопировать из другого блюда» внизу секции.

### КБЖУ

Необязательная секция. Укажите на 100г или на порцию — как вам удобнее:
- **Ккал**
- **Белки, г**
- **Жиры, г**
- **Углеводы, г**

Если заполнено хотя бы одно поле — остальные приравниваются к 0.

### Модификаторы

> Видна, только если включён модуль **Модификаторы**

Здесь вы привязываете к блюду группы опций: размер, прожарку, тип теста и т.д. Покупатель выбирает их при добавлении в корзину.

Подробнее — в статье [Модификаторы](/menu/menu-modifiers).

> **Копирование.** Кнопка «Скопировать из другого блюда» — перенесёт все привязанные группы модификаторов из выбранного блюда.

### Добавки

> Видна для обычных блюд, если включён модуль **Добавки**

Позволяет прикрепить к блюду список дополнительных позиций: соусы, напитки, десерты. Покупатель добавляет их одним кликом при оформлении.

Можно задать индивидуальный максимум добавок для конкретного блюда (если глобальный лимит включён в настройках).

Подробнее — в статье [Добавки](/menu/menu-addons).

### Теги

Цветные значки, которые отображаются поверх фото блюда на витрине. Пример: «🌶 Острое», «🌿 Веганское», «⭐ Новинка».

Выберите нужные теги кликом — они подсветятся цветом. Теги для выбора создаются в разделе [Меню → Теги](/menu/menu-tags).

### Настройки

- **Показывать в меню** — тогл. Если выключен, блюдо скрыто с витрины, но не удалено. Это «стоп-лист»
- **Готовить на кухне** — тогл. Показывается только если включён модуль **Кухня**. Если выключен, блюдо не попадает в очередь кухни

## Редактирование блюда

Кликните по любому блюду в списке — откроется та же форма с заполненными данными. Внесите изменения и нажмите «Сохранить».

## Стоп-лист

Если блюда временно нет в наличии — не удаляйте его. Переключите тогл «Показывать в меню» в выключенное положение прямо из списка блюд. Блюдо исчезнет с витрины и вернётся когда нужно.

## Сортировка

Перетащите блюдо за иконку ручки (⠿) чтобы изменить порядок. Блюда отображаются на витрине именно в этом порядке.

## Привязка к филиалу

Если у вас несколько филиалов — в форме блюда можно задать индивидуальную цену для каждого филиала. Это удобно если цены отличаются по точкам.

## Доступность по филиалам

Если у тенанта несколько филиалов и включён модуль **Филиалы** — в форме блюда появляется мультиселект «**Филиалы**», где можно ограничить позицию конкретными точками. Пусто = блюдо доступно во всех филиалах. Подробнее — в [Филиалы и заведение](/team/team-branches).

## Цены с модификаторами

Цена блюда на витрине — базовая. Каждый модификатор может добавлять к ней сумму (поле «+₽» в форме модификатора). Итоговая цена пересчитывается автоматически.

## Советы

- **Одно блюдо — одно фото.** Покупатели выбирают глазами, фото важнее описания.
- **Короткое название** работает лучше длинного. «Цезарь с курицей» лучше, чем «Салат Цезарь с куриной грудкой, черри и пармезаном».
- **Описание — польза, не состав.** «Сочная говядина с трюфельным маслом» работает лучше, чем «говядина, масло, трюфель».
- **Используйте стоп-лист** вместо удаления. Если блюдо сезонное — выключите его, а не удаляйте.
