# Статусы заказов

Статусы — это этапы обработки заказа. Вы сами создаёте нужные статусы и настраиваете кнопки быстрого перехода между ними. Система не навязывает жёсткий сценарий — вы строите воронку под свои процессы.

## Группы статусов

Каждый статус обязательно относится к одной из четырёх системных групп. Группа определяет поведение заказа:

| Группа | Что означает |
|--------|-------------|
| **Новый** | Заказ только поступил, не обработан |
| **В процессе** | Заказ принят, работа идёт |
| **Выполнен** | Заказ завершён успешно |
| **Отменён** | Заказ не состоялся |

Заказы из групп «Выполнен» и «Отменён» не считаются активными (не попадают в счётчик на дашборде).

> **Пример воронки:** Новый → Принят (в процессе) → Готовится (в процессе) → Готов (в процессе) → Доставляется (в процессе) → Выполнен → Отменён

## Создание статуса

**Заказы → Статусы → «+ Добавить статус»**

### Поля

**Название** *(обязательно)* — то, что увидят и сотрудники (на карточке заказа).
Примеры: «Новый», «Принят», «Готовится», «Готов», «В пути», «Выполнен», «Отменён».

**Группа** *(обязательно)* — выберите из четырёх: Новый / В процессе / Выполнен / Отменён.

**Быстрые действия** — до 2 кнопок перехода, которые появятся на карточке заказа в этом статусе.

## Кнопки быстрого перехода

Это кнопки которые появляются прямо на карточке заказа — сотрудник нажимает и заказ переходит в нужный статус без лишних кликов.

**Пример:** Заказ в статусе «Готов» → кнопка «Передал курьеру» переводит в «В пути».

### Как настроить

В форме редактирования статуса:
1. Нажмите на поле «Быстрое действие 1»
2. Выберите целевой статус из списка
3. Опционально: добавьте «Быстрое действие 2»

Второе действие доступно только если первое уже задано.

## Сортировка статусов

Перетаскивайте статусы за ⠿ чтобы изменить порядок. Порядок влияет на:
- Расположение вкладок на странице заказов
- Порядок в выпадающих списках при смене статуса

## Удаление статуса

Кнопка удаления — только если в этом статусе нет активных заказов.

> **Важно:** Если статус используется как целевой в кухонных настройках (настройки кухни → «Статус при завершении»), сначала отвяжите его там.

## Рекомендуемые воронки

### Доставка + самовывоз

```
Новый → Принят → Готовится → Готов → Выполнен
                                    ↘ В пути → Доставлен
                          ↘ Отменён
```

### Только зал (без доставки)

```
Новый → Принят → Выдан → Оплачен
       ↘ Отменён
```

### С кухонным модулем

Настройте статус «Принят» как **источник для кухни** — при переходе в него блюда автоматически попадут в очередь. Подробнее в статье [Настройки кухни](/kitchen/kitchen-settings).

## Связь с кухней

Статусы плотно связаны с модулем кухни:
- **Статус-источник** — при переходе в него блюда уходят в очередь кухни
- **Статус приготовления** — устанавливается когда повар берёт первое блюдо
- **Статусы завершения** — устанавливаются когда сборщик нажимает «Собрано»

Всё это настраивается в **Кухня → Настройки**.
