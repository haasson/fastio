# Роли и права

> Кастомные роли требуют включения модуля **Кастомные роли** в Настройки → Модули

Роль определяет что сотрудник видит и может делать в панели управления.

## Встроенные роли

| Роль | Что включает |
|------|-------------|
| **Владелец** | Полный доступ ко всему. Нельзя удалить |
| **Менеджер** | Большинство функций кроме биллинга и управления ролями |
| **Сотрудник** | Базовый доступ: заказы, кухня, столы, бронирования |

## Кастомные роли

Если встроенных ролей недостаточно — создайте свои.

**Команда → Роли → «+ Создать роль»**

### Группы прав

Права сгруппированы по разделам. Для каждой группы — переключатель «включить всё» или отдельные чекбоксы.

| Группа | Права |
|--------|-------|
| **Меню** | Просмотр, редактирование, удаление |
| **Заказы** | Просмотр, приём, изменение, смена статуса, отмена |
| **Кухня** | Доступ к очереди, готовка (брать блюда в работу), обзор кухни (для менеджеров) |
| **Столы** | Просмотр, управление |
| **Бронирования** | Просмотр, управление |
| **Записи** | Просмотр, управление, видеть таймлайн всех исполнителей, видеть таймлайн только своего ресурса |
| **Промоакции** | Просмотр, управление |
| **Контент и сайт** | Просмотр, редактирование |
| **Команда** | Просмотр, управление участниками, управление ролями |
| **Настройки** | Просмотр, редактирование |
| **Аналитика** | Просмотр |
| **Биллинг** | Управление |

### Создание роли

1. Введите название (например «Кассир», «Повар», «Менеджер смены»)
2. Выберите нужные права
3. Сохраните

**Пресеты.** Над списком прав — чипы «Начать с пресета» с готовыми наборами под тип бизнеса: для общепита («Повар», «Сборщик», «Курьер», «Официант / Хостес», «Кассир»), для услуг («Мастер», «Старший мастер», «Администратор салона»). Клик по пресету заполняет права и подставляет название (если поле пустое) — дальше можно поправить вручную.

### Дублирование роли

Кнопка «Дублировать» на карточке роли — создаёт копию. Удобно если нужна роль похожая на существующую но с небольшими отличиями.

### Удаление роли

Возможно только если ни один сотрудник её не использует. Если используется — система покажет количество и предложит сначала переназначить сотрудников.

## Типичные конфигурации ролей

**Кассир / Оператор**
- Заказы: просмотр, приём, смена статуса
- Кухня: нет (не нужно видеть очередь)

**Повар**
- Кухня: доступ к очереди + **готовка** (брать блюда в работу)
- Меню: просмотр (чтобы видеть состав блюд)
- Пресет «Повар» собирает этот набор одним кликом

**Сборщик**
- Кухня: доступ к очереди (видит нагрузку, но **без готовки** — не берёт блюда в работу)
- Заказы: просмотр + смена статуса

**Менеджер смены**
- Заказы: всё кроме удаления
- Кухня: очередь + обзор, **но без готовки** — видит нагрузку, блюда в работу не берёт
- Столы: просмотр + управление
- Меню: просмотр (не редактирование)

> **Готовка** (`kitchen.cook`) — отдельное право. Доступ к очереди (`kitchen.view`) лишь показывает нагрузку; брать блюда в работу и отмечать готовыми может только тот, у кого есть «Готовка». Из встроенных ролей её имеют Владелец и Администратор; Менеджеру и Сотруднику назначайте через пресет «Повар» или вручную.

**Контент-менеджер**
- Меню: редактирование
- Контент и сайт: редактирование
- Внешний вид: редактирование

**Мастер / Исполнитель** (онлайн-запись)
- Записи: просмотр, видеть таймлайн только своего ресурса (`view_own`)
- Без права «Видеть таймлайн всех исполнителей» — мастер видит только записи на ресурсах, привязанных к его учётке (поле «Привязка к сотруднику» в карточке исполнителя)
- В сводном списке визитов (с чужими клиентами и телефонами) такой мастер не видит — ему доступен только таймлайн и компактная страница своей услуги
- Чужие записи отрезаются на сервере (RLS) — `select` напрямую через клиент не вернёт чужих

> Дефолтные роли: «Администратор» и «Менеджер» получают `view_all` (видят всё), «Сотрудник» — `view_own` (только своё). Кастомные роли настраивайте вручную.
